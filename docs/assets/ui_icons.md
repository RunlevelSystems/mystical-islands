# 🎨 UI Icons and Documentation Visual Standards
## Purpose
Define visual consistency for markdown docs so pages read like a coherent studio wiki.

## Icon Usage Standards
| Theme | Emoji/Icon | Font Awesome Suggestion | Usage |
| --- | --- | --- | --- |
| Combat | ⚔ | `fa-sword` | abilities, encounters, PvE/PvP |
| Islands/World | 🏝 | `fa-map` | world and region pages |
| Magic | 🧙 | `fa-wand-magic-sparkles` | arcane systems, anomalies |
| Crafting | 🛠 | `fa-hammer` | crafting/build pipelines |
| Pirates/Naval | 🏴‍☠️ / 🚢 | `fa-skull-crossbones`, `fa-ship` | reefs, sea routes, corsair systems |
| Politics | 🏰 | `fa-landmark` | faction, governance, diplomacy |
| Economy | 💰 | `fa-coins` | currency, trade, vendors |
| Building | 🧱 | `fa-trowel-bricks` | claims and settlements |
| Creatures | 🐉 | `fa-dragon` | bestiary and encounters |
| Quests | 📜 | `fa-scroll` | quest chains and objectives |

## Banner Usage Standards
- Use short single-line thematic headers near top of each page.
- Keep banners text-first; avoid large embedded images.
- Use horizontal rules (`---`) to separate major sections.

## Callout Conventions
- `> 🧭` Navigation guidance
- `> ✅` Decisions / standards
- `> ⚠` Risks or implementation cautions
- `> 🧪` Prototype-only notes

## SYNTY-Themed Imagery Usage
- Use compact thumbnail references or links, not full-width large image walls.
- Keep color language consistent with low-poly fantasy style.
- Match region docs to relevant pack families (Kingdom, Pirates, Nature, Dungeon, etc.).

## RPG Symbol Consistency
Always map the same icon to the same system category across all documentation.
